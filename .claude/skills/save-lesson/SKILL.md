# 经验沉淀流程

回顾本次会话,对每条值得留下的经验按下表分流:

| 经验性质                                   | 去处                                       | 例子                           |
| ------------------------------------------ | ------------------------------------------ | ------------------------------ |
| 项目级事实、约定、陷阱(所有协作者都该知道) | CLAUDE.md 对应小节                         | "build 不做类型检查"           |
| 可复用的多步操作流程                       | 新建/更新 `.claude/skills/<名字>/SKILL.md` | 部署步骤、数据迁移流程         |
| 用户偏好、决策背景、时效性上下文           | Claude 持久 memory                         | "结构文档等 refactor 定型再写" |
| git 历史 / 代码本身已能查到的              | 不存,避免重复                              | 某次 bug 的具体修法            |

规则:

- **宁缺毋滥**:每条先查是否已有记录;更新旧条目优于新增。
- CLAUDE.md 保持精炼,过时内容(如分支过渡状态一节)失效后要删除。
- 新 skill 的 frontmatter `description` 必须写清触发时机("当…时使用"),否则不会被调用。
- 沉淀完在回复里列出:写了什么、写到哪了。
